# MMVM网格背包优化清单

## P0 高优先级（先做，收益最大）

- [ ] **O(1) 命中查询**：为格子占用增加 `occupancyOwner`（格子 -> 物品）映射，替代 `GetItemByMask` 的全表遍历。
- [ ] **增量更新占用**：减少 `UpdateMask()` 全量重建；交换流程改为批量提交后统一刷新。
- [ ] **拖拽状态缓存**：仅当 `dragPreviewAnchorPos` 变化时重算 `JudgeFrameBoardState`，同格悬停复用上次结果。
- [ ] **热路径日志开关**：拖拽/交换链路的 `Debug.Log` 改 `#if UNITY_EDITOR` 或统一 `enableDebugLog` 开关。

## P1 中优先级（稳定性 + 可维护性）

- [ ] **拖拽落地一致性**：`EndDrag` 优先使用拖拽阶段最后展示的预览锚点，避免 UI 可换但逻辑拒绝。
- [ ] **交换入口分流**：`TryGetSwapTargetItem` 拆分目标识别和覆盖校验，区分 `LargeToSmall` 与 `SmallToLarge` 的规则。
- [ ] **小换大边界保护**：保留并补齐非锚点触发校验（避免从大物品非锚点进入导致越界错判）。
- [ ] **交换事务化**：`SwapSame/LargeToSmall/SmallToLarge` 统一“失败即返回 + 外层回滚”范式，减少中间态污染。

## P2 低优先级（代码整洁）

- [ ] **命名统一**：统一 `FindSetAtFirst / SetAtFirst / FindPlaceAtFirst` 语义命名。
- [ ] **服务层定位**：明确 `GridPlacementService` 是 Facade（保留监控/日志）还是轻量透传（精简）。
- [ ] **注释规范**：关键规则注释集中在入口函数，避免分支重复解释。

## 回归测试清单（每次修改后必测）

- [ ] 大换小：非锚点完整覆盖可交换；部分覆盖不可交换。
- [ ] 小换大：小物放置位置固定；大物优先回小物旧锚点，不行则找空位。
- [ ] 越界：任意旋转/交换都不允许超出背包边界。
- [ ] UI一致性：吸附框状态与松手结果一致（不出现“显示可换但落地失败”）。
- [ ] 堆叠路径：堆叠成功/失败回滚均正确。
- [ ] 交换失败回滚：物品锚点、UI位置、占用数据三者一致。
